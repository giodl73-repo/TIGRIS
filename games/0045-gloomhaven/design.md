# Gloomhaven — Design (Rules Import)

## Core loop

Each Scenario: Setup (reveal dungeon tiles, place enemies) → Rounds until objective complete or party KO. Each Round: Card Selection (each player chooses 2 cards face-down) → Monster AI Draw (enemy intent card revealed) → Initiative order determined by card initiative numbers → Actions resolve.

## Card hands

Each character starts with a hand of 8–12 cards (varies by class). Each card has: top action (usually offensive/movement) and bottom action (usually support/movement). A player performs one action from each of their 2 selected cards per round. Cards removed from game after use (long rests to recover discard pile).

## Monster AI

Each monster type has an AI deck. Each round, an enemy deck card is drawn revealing: movement target, attack target, special ability. All enemies of that type act identically. Players must predict enemy behavior from the visible AI deck.

## Combat

Attack value + modifier card (drawn from 20-card modifier deck) vs. target shield. Status effects: poison, immobilize, disarm, muddle, curse. Terrain: doors (hidden rooms), traps, obstacles.

## Legacy elements

- Characters retire after completing personal quests → unlock new classes.
- Town events and road events between scenarios.
- Stickers on the campaign map open/close locations.
- 95 scenarios; roughly 80–100 hours campaign.

## Scenario objective types

Kill all enemies, escort, collect items, reach exit, survive X rounds, protect objective.

## Weight profile

Weight 3.86. Setup time 15–25 minutes per scenario. Teach: first scenario is tutorial. Veterans teach new players in 20-30 min; documentation heavy for solo learners.
