# The Crew: Quest for Planet Nine — Rules Summary

## 1. Overview
2–5 players, 20 minutes. Cooperative trick-taking card game. Players work together to complete mission objectives by winning specific tricks. 50 missions form a campaign, each introducing new constraints. Communication is strictly limited: players may give one "communication token" per game (place a card face-up to signal its rank in your hand — top = highest, middle = only, bottom = lowest). No other communication about cards is allowed.

## 2. Key Mechanics
- **Standard trick-taking**: Lead any suit; others must follow suit if able; highest card of led suit wins; trump (rocket suit) beats all.
- **Mission objectives**: Each mission assigns specific task cards to specific players. The player holding a task card must win the trick containing that card. Tasks may have additional constraints (e.g., "Player 1 must win tasks 1, 2, 3 in order").
- **Limited communication (A3)**: The single communication token is the only explicit information-sharing allowed. When played, place a card face-up and indicate if it is your highest/only/lowest card of that suit. This creates mandatory inference from play patterns — if a player ruffs early, they are likely void in the led suit.
- **Mission scaling (A6)**: 50 missions with a consistent rule set but escalating task constraints. Mission 1 requires winning one specified trick; Mission 50 requires coordinated, sequenced task assignments under strict communication blackouts. Teach the base game; missions teach themselves.

## 3. Cooperative Structure
All players succeed or fail together. If any player wins a trick they were not supposed to win a task-requiring card in, the mission fails. This creates strict interdependency: every trick outcome matters for every player.

## 4. Contrast with Furrow (#28)
FURROW uses trick-taking as competitive farming action-selection (B5 architectural novelty). The Crew uses trick-taking as cooperative mission-fulfillment with communication constraints (A6 teachability + A3 mandatory coordination). The mechanism is shared; the structural application is orthogonal.
