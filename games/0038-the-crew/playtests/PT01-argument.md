# PT01 — THE CREW: Argument Record (condensed)

## Key argument moments

**A6 (Stegmaier anchor) — Defend**: "Mission 7: Players have never failed a 4-player mission. They succeed. Mission 8 introduces a sequencing constraint: Player 2 must win Task A before Player 3 wins Task B. The mission teaches sequencing without a new rules page — the constraint is on the task card. By Mission 15, players are coordinating multi-step sequences through a single communication token and positional inference. The 50-mission arc is a structured learning progression embedded in play, not in a rulebook. A6 earns as a Stonemaier-paradigm teachability instance."

**A3 (Chvátil) — Defend**: "Trick 4, Mission 9: P1 leads hearts. P2 ruffs with a rocket (trump). This is the only time P2 would ruff a heart in normal play — if P2 is not void in hearts, this is a wasteful use of trump. The panel infers P2 is void in hearts. P3 now knows not to lead hearts toward P2. The communication constraint FORCES this inference-based interaction: because explicit communication is banned (token already used), every action is an information signal. A3 fires on mandatory indirect interaction — stronger than most area-control games because the inference chain is tighter. A3 earns."

**A1 (Knizia) — Defend**: "Three rules (lead any suit, follow suit if able, rockets trump) plus one communication token produce 50 missions of emergent cooperative complexity. The elegance is verifiable: the game teaches in 3 minutes; Mission 50 takes 40 minutes of concentrated play. The rules-to-depth ratio is extreme. A1 earns."

### A6↔A3 COLLISION (OP, 5-3)

**Stegmaier (A6)**: "A6 asks: 'does the game's structure teach cooperation progressively?' — the 50-mission arc is the clearest positive example in 38 TIGRIS games. A3 asks: 'does the game force meaningful interaction?' — mandatory communication constraints do force team coordination on every trick. These are different registers: A6 fires on the DESIGN ARCHITECTURE of the learning curve (how the game is structured to teach); A3 fires on the INTERACTION DENSITY of each play session (how players must engage with each other). A game can teach cooperation via mission structure AND enforce interaction via communication constraints."

**Chvátil (A3)**: "A3 counter: in most trick-taking, each player acts independently per trick. The Crew's communication constraint converts trick-taking from an individual action into a team inference problem. This is not just teachability — this is A3 as mandatory team reasoning per trick. The communication token is not a teaching tool; it is an interaction mechanic."

**Vote: 5-3 OP.** Both A6 and A3 earn. Mission structure teaches (A6, design architecture, campaign-level); communication constraints force interaction (A3, play mechanics, trick-level). Different registers — 16th temporal-register OP.
