# CHORUS — Concept

## Premise

CHORUS is a 2–6 player simultaneous-action cooperative scoring game where ALL player count variants use identical rules. Each player is a "Voice" contributing to a shared harmonic score. The game is architecturally designed to be equally good at every count via a modular Voice system: at 2p each player controls 3 Voices; at 4p each player controls 2 Voices; at 6p each player controls 1 Voice. Identical rules across all counts; only the number of Voices per player changes.

D3 anchors because making the game architecturally scale from 2 to 6 without rule changes is the explicit design challenge. This is not just "works at all counts" — it's "engineered to be equally good at all counts."

## Anchor mechanic

**D3 Count-Robustness.** The modular Voice system: each Voice has one action per round (Compose, Conduct, Sustain, Rest). At 2p, each player manages 3 Voices independently (complex individual management). At 6p, each player manages 1 Voice (coordination with full group). At 4p, 2 Voices each (middle ground). The game's strategic space scales with the Voice distribution — identical rules, different coordination challenge at each count.

## Target earn-candidates

- **D3** (anchor; Stegmaier) — identical rules 2–6; modular Voice system
- **A6** — taught in 15 min; Voice/action system is thematically grounded
- **A3** — cooperative = mandatory interaction; harmony scoring requires coordination
- **A5** — simultaneous action = near-zero downtime
- **C1** — Harmony tokens limited; perpetual scarcity
- **A2** — per-Voice action selection; multi-Voice management density
- **A4** — die/card draw variance in Harmony resolution
- **C7** — 4 clear actions per Voice
- **A1** — minimal rules; complete cooperative game
- **D1** — family-accessible at 6p; deep at 2p

## Collision candidate

**D3↔A5 OP** — count-robustness (D3: the structural design achievement of identical rules at all counts) vs. simultaneous-action pacing (A5: the zero-downtime mechanism that makes scaling achievable). Different registers: D3 = the design goal; A5 = the mechanism that enables it.
