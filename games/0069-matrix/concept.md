# MATRIX — Concept

## Premise

MATRIX is a 2–4 player deck-and-die game where players build personal 4×4 matrices of cards. Each round, players roll a shared pool of dice; dice results activate corresponding rows AND columns on the matrix simultaneously, creating intersection bonuses where an activated row meets an activated column.

The key A4 claim: players build their die pool over the first 3 rounds, calibrating which dice are present in the pool to control which row/column combinations will be activated. This is variance management via die selection — building the right die pool for your matrix configuration.

The A2 secondary: each card placement in the matrix is a multi-axis decision (row bonus, column bonus, intersection potential with current and future die composition).

## Anchor mechanic

**A4 Variance Calibration.** Die-pool construction determines the probability distribution of which matrix positions activate each round. A die that produces results 1-4 activates rows 1-4 (top half of matrix); a die that produces results 5-8 activates rows 5-8. Players select dice to add to their pool based on their matrix configuration — calibrating variance to their built positions.

**A2 secondary** — per-placement matrix optimization: where to place each card to maximize row bonus, column bonus, and intersection bonus potential given current die pool.

**B3 secondary** — die→row→column→intersection cascade: a single die roll activates a row, which activates a column (via intersection bonus), which may trigger a cascade bonus from adjacent intersection tokens. Depth-3 chain from a single roll.
